# Trazabilidad de Objetivos (RA/CE) - ASIR

Este proyecto está diseñado para cubrir los siguientes puntos del currículo oficial:

| Bloque Técnico | Tarea Específica | RA / CE Cubierto |
| :--- | :--- | :--- |
| **Infraestructura** | Uso de Docker para PostgreSQL | RA1 (d, e, f) |
| **Diseño Lógico** | Modelo ER y Normalización (3FN) | RA2 (a, c, d, e, f, g, h) |
| **Diseño Físico** | Creación de tablas y Tipos de Datos | RA3 (a, b, c, d, h) |
| **Integridad** | Restricciones `CHECK` (DNI, IBAN...) | RA3 (e, i) |
| **Explotación** | Consultas simples, Joins y Subconsultas | RA4 (b, c, d, e, f) |
| **Manipulación** | Inserts, Updates y Soft Delete | RA5 (b, d, e) |
| **Seguridad** | Gestión de Roles y Privilegios (RBAC) | RA5 (e) + RA6 (g) |
| **Seguridad** | Backups automáticos y restauración | RA6 (a, b, c) |
| **IA** | Auditoría y validación de código | Transversal (Calidad y Ética) |

## Glosario de RAs
*   **RA1:** Reconoce elementos de bases de datos.
*   **RA2:** Diseña modelos lógicos normalizados.
*   **RA3:** Realiza diseño físico (DDL).
*   **RA4:** Consulta información (SELECT).
*   **RA5:** Modifica información (DML).
*   **RA6:** Aseguramiento de la información (Backups/Seguridad).
